# MOPS 即時重大資訊監控 AI Agent 使用指南

## 概述

本文檔說明如何使用新訓練的 **MOPS AI Agent** 來監控台灣 MOPS（公開資訊觀測站）的「即時重大資訊」。

### 架構
- **Agent 1 (Monitor)**: `mops_monitor_agent.py` - 解析首頁即時重大資訊表格
- **Agent 2 (Diff)**: `mops_diff_agent.py` - 比較快照並生成差異報告
- **整合**: `site_profiles.py` + `change_agent.py` - 與系統整合

---

## 功能說明

### Agent 1: 即時重大資訊監控 (`mops_monitor_agent.py`)

負責從 MOPS 網站首頁提取「即時重大資訊」表格資料。

**提取內容:**
- 公司代碼 (e.g., 2330, 1101)
- 公司簡稱 (e.g., 台積電, 台泥)
- 發布時間 (e.g., 2026/04/28 14:30)
- 訊息標題 (e.g., 董事長異動公告)
- 詳細連結

**核心函數:**
```python
extract_mops_structured(html: str, source_url: str) -> dict
```
- 輸入：網頁 HTML 與來源網址
- 輸出：結構化資料（含公司代碼、名稱、時間、標題、連結）

**快照格式示例:**
```
[站點] 公開資訊觀測站 MOPS
[區塊] 首頁 > 即時重大資訊
[來源] https://mops.twse.com.tw/mops/web/t05sr01_1
[筆數] 5
[即時資訊列表]
[2330] | 台積電 | 2026/04/28 14:30 | 董事長異動公告 | https://...
[1101] | 台泥 | 2026/04/28 14:25 | 財務報告公告 | https://...
```

---

### Agent 2: 差異報告與摘要 (`mops_diff_agent.py`)

負責比較兩個時間點的快照，識別新增/移除的訊息，並生成通知報告。

**核心功能:**
1. **新增項目偵測**: 找出當前快照中前一個快照沒有的項目
2. **移除項目偵測**: 找出前一個快照中當前快照沒有的項目
3. **AI 增強摘要** (可選): 使用 Gemini 生成更優化的通知文字
4. **基本摘要**: 無 API 時自動降級為結構化摘要

**核心函數:**
```python
generate_mops_diff_report(
    *,
    previous_snapshot: str | None,
    current_snapshot: str,
    api_key: str | None = None,
    model_name: str | None = None,
) -> str | None
```

**報告示例:**
```
MOPS 即時重大資訊更新｜首頁 > 即時重大資訊
本次新增：2 筆
本次移除：1 筆
----------------------------------------
新增即時重大資訊：
  1. [2330] 台積電 (2026/04/28 14:35) | 重大資產收購公告
  2. [2412] 中華電 (2026/04/28 14:12) | 股東常會開催
----------------------------------------
移除資訊：
  - [6005] 群益證 (2026/04/28 14:10) | 公司消息
```

## 系統整合

### 1. 自動路由（在 `change_agent.py` 中）

系統會自動識別 MOPS URL 並調用相應的 Agent：

```python
if is_mops_realtime_url(url):
    mops_report = generate_mops_diff_report(...)
```

### 2. 網站快照提取（在 `site_profiles.py` 中）

自動提取已知網站的結構化內容：

```python
if host.endswith("mops.twse.com.tw") and "t05sr01_1" in (parsed.path or ""):
    return extract_mops_snapshot(url=url, html=html, full_text=full_text)
```

## 使用流程

### 方法 1: 透過 Web UI 新增訂閱

1. 開啟系統：`http://127.0.0.1:5000`
2. 登入 / 註冊帳號
3. 進入「我的追蹤網站」
4. 點「新增」，填入：
   - **URL**: `https://mops.twse.com.tw/mops/web/t05sr01_1`
   - **名稱**: `MOPS 即時重大資訊`
   - **觀看描述** (可選):
     ```
     追蹤首頁即時重大資訊表格中的新增/更新訊息，
     特別關注科技業（台積電、聯發科）和金融業的公告。
     ```
   - **檢查頻率**: 建議 30~60 分鐘一次（可根據需要調整）

5. 保存並等待第一次自動檢查
6. 有更新時，會在「我的追蹤網站」顯示「差異摘要」

```bash
# 執行測試腳本
python test_mops_agent.py
```

輸出會顯示：
- 項目提取狀況
- 差異偵測結果

## 配置選項

### 環境變數

**`.env` 檔案中的相關設定:**

```env
# MOPS 檢查間隔（分鐘）
# 建議值：30 (30 分鐘檢查一次)
# 或     60 (1 小時檢查一次)
CHECK_INTERVAL_MINUTES=60

# Gemini API Key（可選，用於 AI 增強摘要）
GEMINI_API_KEY=your_api_key_here

# AI 模型選擇（預設：gemini-1.5-flash）
AI_SUMMARY_MODEL=gemini-1.5-flash
```

### 自訂「觀看描述」

在新增訂閱時填入詳細的觀看描述，可幫助 AI 更精確地過濾和摘要：

**好例子：**
```
監控上市公司重大資訊：
1. 特別關注台積電、聯發科、鴻海的董事會/股東會公告
2. 注意財務報表公告日期
3. 重大訊息變化（合併、收購、重組）
```

**較泛的例子：**
```
追蹤所有新增的重大資訊
```

## 常見問題 (FAQ)

### Q1: 為什麼沒有偵測到新增?
**A:** 
- 檢查 URL 是否正確: `https://mops.twse.com.tw/mops/web/t05sr01_1`
- 確認頁面載入完整（可能需要 JavaScript 渲染，建議使用系統自動檢查）
- 查看快照是否正確提取（在資料庫檢查 `snapshot` 欄位）

### Q2: 如何加快檢查速度?
**A:** 在新增訂閱時設定較短的檢查間隔：
- 開發/測試: 5~10 分鐘
- 一般監控: 30~60 分鐘
- 非實時: 120~1440 分鐘

### Q3: Gemini API Key 的作用?
**A:** 
- **有 API Key**: 使用 AI 生成更優化、更自然的通知文字
- **無 API Key**: 自動使用結構化摘要（仍然可用，但較為呆板）

### Q4: 可以監控特定公司嗎?
**A:** 
在「觀看描述」中明確列出公司名稱/代碼，系統會優先突顯相關項目：
```
監控：台積電(2330), 聯發科(2454), 鴻海(2317)
```

### Q5: 如何匯出或分享報告?
**A:** 
在「我的追蹤網站」點選訂閱，可：
- 查看完整快照與差異
- 複製通知文字
- 導出為 CSV (日後功能)

## 技術細節

### 資料流程

```
網頁 HTML
   ↓
mops_monitor_agent.extract_mops_structured()
   ↓
結構化快照文字
   ↓
系統儲存 (資料庫)
   ↓
[下次檢查時]
   ↓
前一快照 vs 當前快照
   ↓
mops_diff_agent.generate_mops_diff_report()
   ↓
差異報告 (帶 AI 增強 或 基本摘要)
   ↓
通知使用者
```

### 去重機制

項目唯一性由以下優先順序決定：
1. 完整 URL (如果有效)
2. 公司代碼 + 發布時間 + 標題 組合
