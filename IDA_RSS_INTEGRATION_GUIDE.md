## 🔄 更新通知流程

### 系統如何工作:

1. **定期檢查** (預設每 6 小時)
   - 平台自動連線到 RSS feed
   - 比較最新內容與上次儲存的版本

2. **偵測變更**
   - 使用 SHA-256 hash 快速判斷是否有新聞
   - 忽略小於 3 字符的微小變更

3. **生成差異**
   - 使用 `diff_match_patch` 庫比對新舊內容
   - 提取新增的新聞項目 (通常是最新的 1-5 則)
   - 提取移除或修改的項目

4. **發送通知** (給所有訂閱者)
   - **應用內通知**: 在儀表板顯示，包含差異摘要
   - **電子郵件通知** (如已設定): 詳細的更新報告
   - **狀態**: 標記為未讀，用戶可以查看

### 通知訊息範例:
```
新聞發布有更新：
【新增】
- 為產業請命 經濟部呼籲維持現行空污法制度...
- 以 AI 感測晶片與抗噪技術，為智慧座艙注入靈魂

【更新時間】
2026-04-28 14:30:00
```

## 🔧 故障排除

### 問題 1: 沒有收到通知

**可能原因:**
1. 檢查頻率設定過高，導致錯過更新
2. RSS 服務暫時無法連線
3. 郵件設定未正確配置

**解決方案:**
1. 在訂閱詳情中點擊 **"立即檢查"** 進行手動驗證
2. 查看應用日誌中是否有錯誤信息
3. 檢查 SMTP 郵件設定

### 問題 2: 通知重複

**可能原因:**
- RSS feed 本身的內容波動 (例如新聞編輯)

**解決方案:**
- 這是正常的；系統使用內容 hash 判斷真正的變更
- 微小編輯不會觸發重複通知
